合并分支
如：将dev分支合并到master上
1.查询分支
git branch
2.转到master上
git checkout master
3.进行合并
git merge dev -m "进行合并"

克隆远程分支
git clone -b gh-pages [地址]

版本回退
1.查询版本
git log --pretty=oneline
2.回退到制定版本
git reset --hard [版本号前6、7个字符]
3.更新回退命令到远程（可选）
git push -f origin [分支]

新建仓库
1.本地初始化
git init
2.添加本地文件
git add .
3.创建版本
git commit -m "说明"
4.在github创建新的仓库并拿到地址
5.本地仓库链接远程仓库
git remote add origin [地址]
6.推送到远程仓库
git push origin master


Atom

Atom软件安装完第一次使用时中间会有一条白线。
这条白线是用于指示内容超出该线后在网页上的排版可能会和Atom显示的不一样。

默认情况下，该线与左边的距离为615px。

比如现在我这个内容元素的宽度为800px。为了减少调试，我要把改线的距离也变成800px。

打开File->Stylesheet，在下面添加代码：

atom-text-editor .wrap-guide{
  border: 1px solid gray;
  transform: translateX(-615px);
  margin-left: 750px;
}

这里的750px是因为Atom软件里面的字体大小跟页面的不一样，所以减少一点。

如果要隐藏该线，只需要在里面添加：

atom-text-editor .wrap-guide{
  visibility: hidden;
}

Egret

数据容器中的皮肤需要提前加载，否则在使用时候在加载的话会导致加载过慢，属性没有附上。

有时候报某一个控件未定义错误，很大可能是因为皮肤没有加载完成引起的。这时候要做一个初始化方法，判定控件时候初始化完成。

在监听touch_begin、touch_end、touch_move的事件中，可能因为移动太快，move事件跟不上。
在抬起的时候会触发TOUCH_RELEASE_OUTSIDE事件，导致touch_begin、touch_end不能成对触发。

在获取舞台宽、高的时候，直接通过this.stage.width/height获取到的是零。
需要通过egret.MainContext.instance.stage来获取。

遇到页面没有正常调用方法、没有正常开启或关闭的时候，先想想有没有可能是页面还没有初始化完毕。

储存信息到本地

// 存储
var key:string = "bestscore";
var value:string = "95";
egret.localStorage.setItem(key,value);

// 读取
var score:string = egret.localStorage.getItem(key);

// 删除
egret.localStorage.removeItem(key);

// 清空所有数据
egret.localStorage.removeItem(key);
