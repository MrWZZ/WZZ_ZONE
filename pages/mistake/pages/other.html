Git


个人博客

丢弃本地所有修改（新增、删除、更改）
git checkout .

合并分支
如：将dev分支合并到master上
1.查询分支
git branch
2.转到master上
git checkout master
3.进行合并
git merge dev -m "进行合并"

克隆远程分支
git clone -b gh-pages [地址]

版本回退
1.查询版本
git log --pretty=oneline
2.回退到制定版本
git reset --hard [版本号前6、7个字符]
3.更新回退命令到远程（可选）
git push -f origin [分支]

新建仓库
1.本地初始化
git init
2.添加本地文件
git add .
3.创建版本
git commit -m "说明"
4.在github创建新的仓库并拿到地址
5.本地仓库链接远程仓库
git remote add origin [地址]
6.推送到远程仓库
git push origin master

在克隆主分支后获取其他分支
1.克隆主分支
git clone [地址]
2.查看远程分支
git branch -a
3.遥控远程分支
git checkout -b [本地起名(随意)] origin/[分支名] 

amend操作
amend在于提交的时候不创建新的节点，而是覆盖上一个节点。
如果在amend操作之后再加上no-edit的话，就不修改提交的说明。
完整外码: git commit --amend --no-edit
这时候在推送就需要强制推送。
git push -f



Atom

Atom软件安装完第一次使用时中间会有一条白线。
这条白线是用于指示内容超出该线后在网页上的排版可能会和Atom显示的不一样。

默认情况下，该线与左边的距离为615px。

比如现在我这个内容元素的宽度为800px。为了减少调试，我要把改线的距离也变成800px。

打开File->Stylesheet，在下面添加代码：

atom-text-editor .wrap-guide{
  border: 1px solid gray;
  transform: translateX(-615px);
  margin-left: 750px;
}

这里的750px是因为Atom软件里面的字体大小跟页面的不一样，所以减少一点。

如果要隐藏该线，只需要在里面添加：

atom-text-editor .wrap-guide{
  visibility: hidden;
}
