思考工厂模式

  本质：选择实现。
  简单工厂的目的在于为客服端来选择相应的实现，从而使得客服端和实现实现之间解耦。

  如何实现选择
  + 静态传参：读取配置文件；直接传入参数。
  + 动态传参：运行期间读取内存。

  何时选用简单工厂
  + 要完全封装隔离具体实现，让外部只能通过借口来操作封装体。
    让客户端通过工厂来获取相应的接口，而无需关心具体的实现。

  + 要把对外创建对象的职责及综合那个管理和控制。

  
优缺点

  优点
  + 帮助封装。
  + 解耦。

  缺点
  + 可能增加客户端的复杂度。
  + 不方便扩展子工厂。

  
命名规范

  + 类名建议为“模块名称+Factory”。
  + 方法名称通常为“get+接口名称”或者是“create+接口名称”。

代码示例

接口

  //接口
  public interface Api
  {
    public void operation(String s);
  }

  //实现A
  public class ImplA implements Api
  {
    public void operation(String s)
    {
      //...
    }
  }

  //实现B
  public class ImplB implements Api
  {
    public void operation(String s)
    {
      //...
    }
  }


工厂

  public class Factory
  {
    //选择方式：*1.传参；2。反射；3.读取配置文件。
    public atatic Api createApi(int type)
    {
      Api api = null;
      if(type == 1)
      {
        api = new ImplA();
      }
      else if(type == 2)
      {
        api = new ImplB();
      }
      return api;
    }
  }


客户端

  public class Client
  {
    public static void main(String[] args)
    {
      Api api = Factory.createApi();
      api.operation("hello");
    }
  }
