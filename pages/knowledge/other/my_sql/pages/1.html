InnoDB储存引擎
提供了良好的事务管理、崩溃修复能力和并发控制。
缺点是读写效率稍差，占用的数据空间相对比较大。

MyISAM储存引擎
优势在于占用空间小，处理速度快。
缺点是不支持事务的完整性和并发性。

MEMORY储存引擎
处理数度非常快，但是数据容易丢失，生命周期短。


数据类型


数字类型

整型数据类型
+ TINYINT             1字节
+ BIT                 1字节
+ BOOL                1字节
+ SMALLINT            2字节
+ MEDIUMINT           3字节
+ INT                 4字节
+ BIGINT              8字节

浮点型数据类型
+ FLOAT               8/4字节
+ DOUBLE              8字节
+ DECIMAL             自定义长度

多学两招
在创建表时，使用哪种数字类型应该遵循以下原则：
+ 选择最小的可用类型，如果数值永远不超过127，则使用TINYINT比INT强。
+ 对于完全都是数字的，可以选择整数类型。
+ 浮点类型用于可能具有小数部分的数，例如货物单价、网上购物交付金额等。


字符串类型

常规字符串类型
+ [national]                                          固定长度为M的字符串，其中M的取值范围为0~255。
  char(M)                       0~255个字符            national关键字指定了应该使用默认的字符集。
  [binary|ASCII|unicode]                              binary关键字指定了数据是否区分大小写（默认区分）。
                                                      ASCII、unicode指定了该列应该使用的字符集。

+ char                          0~255个字符           同上

+ [national]
  varchar(M)                    0~255个字符           长度可变，其他同上。
  [binary]

TEXT和BLOBL类型
这两种类型，他们的大小可以改变，TEXT类型合适储存长文本，而BLOB类型适合储存二进制数据（如声音、图像等）。

+ TINYBLOB           2^8~1(字节)
+ TINYTEXT           2^8~1
+ BLOB               ..16..
+ TEXT               ..16..
+ MEDIUMBLOB         ..24..
+ MEDIUMTEXT         ..24..
+ LONGBLOB           ..32..
+ LONGTEXT           ..32..

ENUM和SET类型
+ Enum("value1","value2",...)      65535      该类型的列只可以容纳所列值之一或为NULL
+ Set("value1","value2",...)       64         该类型的列可以容纳一组值或为NULL

多学两招
在创建表和使用字符串类型时应遵循一下原则：
+ 从速度方面考虑，要选择固定的列，可以使用CHAR类型。
+ 要节省空间，使用动态的列，可以使用VARCHAR类型。
+ 要将列中的内容限制在一种选择，可以使用ENUM类型。
+ 允许在一个列中有多余一个的条目，可以使用SET类型。
+ 如果要搜索的内容不区分大小写，可以使用BLOB类型。
